# 02. 权限配置：让本地 Codex 直接写飞书文档 / 多维表格

这一篇只讲怎么把权限配通。

前提是前一篇里的 bridge 已经跑起来了，你已经可以在飞书里和本地 Codex 对话。  
接下来做的是另一件事：把 `lark-cli` 授权成用户身份，让本地 Codex 能直接读写飞书文档，后面再继续扩到多维表格。

## 一、先装好 `lark-cli`

如果前面已经装过，这一步可以跳过。

```bash
npx @larksuite/cli@latest install
```

## 二、先发起用户身份授权

这一步是在终端里做，不是在飞书聊天框里做。

先执行：

```bash
lark-cli auth login --domain docs --domain drive --domain wiki --no-wait --json
```

这条命令会返回两样关键东西：

1. `verification_url`
2. `device_code`

这里的作用很简单：

- `verification_url` 用来打开授权页
- `device_code` 留给终端最后收尾

## 三、把授权链接转成二维码，再用手机扫

拿到 `verification_url` 以后，可以继续生成二维码给手机扫。

我当时就是终端先拿授权链接，再把授权链接转成二维码发出来给手机扫。

这里要注意一个细节：

如果你在 Windows 下直接走 `.cmd` 包装层，URL 里有 `&` 时很容易被拆坏。  
更稳的做法是直接调用真正的 `lark-cli.exe`，不要让 `.cmd` 去解析带 `&` 的 URL。

## 四、手机上完成授权

扫完以后，手机上看到的就是飞书开放平台授权页。

![飞书开放平台授权页](../assets/04-feishu-auth.jpg)

这一步直接按页面顺序走就行：

1. 确认应用没错
2. 确认权限范围
3. 点“开通并授权”

图里这一页其实已经把重点写得很清楚了，像：

- 查看 / 编辑新版文档
- 查看云空间文件元数据
- 开通多维表格、电子表格、幻灯片的常用权限

如果这里没有勾到，后面本地命令也不会自动有权限。

## 五、手机授权完以后，回终端收尾

手机上点完“开通并授权”以后，还没结束。

终端里还要再执行一次：

```bash
lark-cli auth login --device-code <device_code>
```

这里的 `<device_code>` 就是第二步那条命令返回的那个值。

执行完这一步，本地这套用户授权才算真正写进 `lark-cli` 配置。

## 六、先检查当前身份是不是对的

授权完成后，不要急着直接写文档，先检查状态。

```bash
lark-cli whoami
lark-cli auth status --json --verify
```

主要看三样：

1. `identity` 是不是 `user`
2. token 是否有效
3. scope 是否正常

如果这里不是 `user`，后面就不要继续做文档写入测试。

## 七、先读文档

先拿一个已经存在的飞书文档或 wiki 链接做读取测试：

```bash
lark-cli docs +fetch --doc "<你的飞书文档链接>" --as user
```

这一步确认的是：

1. 当前身份能不能读这个文档
2. 当前目标文档有没有选错

## 八、再做写入测试

不要一上来就全量覆盖正式文档。

更稳的顺序是：

1. 先 dry-run
2. 再追加一小段测试内容
3. 再读回去确认
4. 最后把测试段落删掉

这样不会把正式文档改乱，也方便排查到底是：

- 写入参数不对
- 当前身份不对
- 还是目标文档权限不够

## 九、文档写通以后，再继续做多维表格

多维表格不用另起一套权限思路。

文档先打通以后，后面继续接多维表格，复用的还是这套东西：

1. 同一个飞书应用
2. 同一套用户身份授权
3. 同一个本地 Codex
4. 同一个 `lark-cli`

区别只是后面补的目标从“文档”换成了“表”。
